# OmniServe (Edge ML FastAPI Backend) 🚀⚙️

A high-performance, production-ready REST API built to serve Python logic and Machine Learning workflows over local HTTP endpoints. 

This project demonstrates the critical MLOps step of taking raw, isolated Python scripts (in this case, code generated by an autonomous AI swarm) and wrapping them in a scalable backend microservice so they can be consumed by front-end applications, mobile apps, or other servers.

## ✨ Key Features

* **Lightning Fast Routing:** Utilizes FastAPI, one of the fastest Python web frameworks available, to route requests with minimal latency.
* **RESTful Architecture:** Built with standard HTTP GET/POST methodologies for easy integration with modern tech stacks.
* **Interactive Documentation:** Automatically generates a live Swagger UI dashboard (`/docs`), allowing developers to test endpoints, inject variables, and view JSON responses natively in the browser without requiring Postman.
* **Microservice Ready:** Container-agnostic and ready to be deployed to cloud services (AWS, GCP) or run locally on edge hardware.

## 🛠️ Tech Stack

* **Backend Framework:** FastAPI
* **ASGI Web Server:** Uvicorn
* **Language:** Python 3.10+
* **Data Formatting:** JSON / Pydantic

## 🚀 Setup & Installation

**1. Install Dependencies**
Ensure you have Python installed, then run the following to install the core backend engine:

    pip install fastapi uvicorn

**2. Boot the Server**
Navigate to the project directory and start the Uvicorn ASGI server with live-reloading enabled:

    uvicorn main:app --reload

## 💻 Usage & API Endpoints

Once the server is running (defaulting to `http://localhost:8000`), you can access the following endpoints:

* **Health Check (GET `/`)**
  Returns the server status and confirms the API is online.
  
* **Logic Endpoint (GET `/api/fibonacci/{number}`)**
  Dynamically calculates the requested sequence using the AI-generated logic.
  *Example Response:*
  `{"requested_n": 10, "fibonacci_result": 55, "status": "success"}`

* **Swagger UI (GET `/docs`)**
  Access the interactive developer dashboard to test endpoints natively.
